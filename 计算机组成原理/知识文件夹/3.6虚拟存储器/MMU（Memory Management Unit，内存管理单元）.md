### **MMU是 CPU 内部专门负责地址翻译的硬件模块，是虚拟存储系统能够运行的物理基础。**

---

## **MMU 在整个体系中的位置**

前面讲的所有地址翻译过程——拆分虚拟地址、查 TLB、遍历页表、拼接物理地址、权限检查——这些事情不是软件做的，而是由 MMU 这块硬件**自动完成**的，整个过程对程序完全透明。

```
CPU 核心
    │
    │ 发出虚拟地址 VA
    ▼
┌─────────────────────────────┐
│           MMU               │
│  ┌───────┐   ┌───────────┐  │
│  │  TLB  │   │页表遍历器  │  │
│  └───────┘   └───────────┘  │
│     地址翻译 + 权限检查       │
└─────────────────────────────┘
    │
    │ 输出物理地址 PA
    ▼
 Cache / 主存
```

MMU 是 TLB 和页表遍历器（Page Table Walker）的"容器"，它们都是 MMU 内部的组成部分。

---

## **MMU 具体做哪些事**

**地址翻译**是最核心的职责，也就是把 CPU 发出的虚拟地址转换成物理地址

**权限检查**是另一个重要职责。每次地址翻译时，MMU 会同时检查页表项里的权限位（RWX），如果程序试图写一个只读页（比如代码段），或者执行一个不可执行的页（比如数据段），MMU 会立刻触发异常，阻止这次非法访问。这是操作系统内存保护机制的硬件基础。

**TLB 管理**也由 MMU 负责，包括 TLB 的查找、命中判断、未命中时触发页表遍历、以及将新的映射填入 TLB。
