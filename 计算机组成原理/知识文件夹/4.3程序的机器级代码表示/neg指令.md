### `neg` 指令计算的是补码（二进制补码），不是反码。

这是一个很常见的概念混淆，区分一下：

- **反码（One's Complement）**：将每一位取反，0变1、1变0。例如 `eax = 0000_0001`，反码是 `1111_1110`。
- **补码（Two's Complement）**：反码 + 1。例如 `eax = 0000_0001`，补码是 `1111_1111`，即 `-1` 的二进制表示。

`neg` 指令执行的操作是 `结果 = 0 - 操作数`，也就是数学意义上的取负值。在计算机的有符号整数体系中，一个数的负值正是用**补码**来表示的。

以 `eax = 1`（即 `0x00000001`）为例：

- `neg eax` 执行后，`eax = -1`，其二进制表示为 `0xFFFFFFFF`
- 而 `1` 的反码是 `0xFFFFFFFE`，并不等于 `-1`

所以图中写的 `eax ← -eax`，这里的 `-eax` 指的是 **算术取负**，其底层实现正是补码运算，书中表述完全正确。
